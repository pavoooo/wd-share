HTML(文本标记语言)

Html结构

  • 页面的基本结构(html4页面结构/html5页面结构)
  • 块级元素和行内元素
    • 常见的块级元素(block)：div, p, table, ul, ol, li, h1-h6, header, section, article等
    • 常见的行内元素(inline)：span, a, label, input, i, em, strong等
  • 几条原则
元素类型 块级元素 行内元素
特性 独占一行，默认情况下，其宽度自动填满其父元素宽度 不会独占一行，相邻行内元素会排列在同一行里，直到一行排不下，才会换行，其宽度随元素的内容而变化
width、height属性 可以设置，设置了宽度还是独占一行 无效
margin和padding属性 可以设置 水平方向的padding-left、padding-right、margin-left、margin-right都产生边距效果，但竖直方向的padding-top、padding-bottom、margin-top、margin-bottom却不会产生边距效果。
内联元素和块状元素

  • 双标签一定要闭合
  • 尽量不要给标签设置属性值，让css进行处理(标签和属性分离)
  • 标签语义划
  • 注意BFC（块级上下文格式）

盒模型

所有HTML元素可以看作盒子，CSS盒模型本质上是一个盒子，封装周围的HTML元素，它包括：边距，边框，填充，和实际内容。

  • W3C标准和模型 
    总宽度= width + margin(左右) + padding(左右) + border(左右)
  • IE盒模型(怪异盒模型) 
    总宽度= width + margin(左右)（即width已经包含了padding和border值）
盒子模型

两个概念

  • BFC(块格式化上下文) 
    建立一个隔离的空间，断绝空间内外元素间相互的作用。
  • 文档流 
    从头到尾按照文档的顺序，该在什么位置就在什么位置，自上而下，自左到右的顺序

  • 右浮动

float

定位

  • static,relative,absolute,fixed
  • absolute和fixed会使元素变成块元素
  • absolute的元素的位置相对于最近的已定位(position的值为非static)祖先元素,默认是body。fixed的元素其定位元素永远是body。
  • 元素同样会脱离文档流
